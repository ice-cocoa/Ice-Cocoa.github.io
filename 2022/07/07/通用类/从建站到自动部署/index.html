从建站到自动部署

发表于 2022-07-07 更新于 2022-07-15 分类于 通用类

使用本主题发帖的基本说明，包括创建帖子文件，编辑帖子内容，本地测试发布效果，正式发布帖子，自动部署。万一忘了，或者哪天需要新建网站的时候再看看。

第一类发布方式

使用VScode编辑帖子内容，代码提交到github的博客代码仓库，利用工作流自动部署到github的博客页面仓库。
具体流程如下：

配置github，克隆代码到本地

  1. 登录github网站，创建账号

  2. 创建名为”MyBlogCode”的私人仓库，这个仓库保存了相关代码和帖子;这个仓库名字可以自己起

  3. 创建名为”你的用户名.github.io”的公开仓库，这个仓库保存了博客页面

  4. 安装Git工具，这个工具能够本地代码直接上传到仓库，或者从仓库拉去代码

  5. 安装好Git工具后，右击桌面空白处，点击”Git Bash here”,进行以下配置

     git config --global user.name "github用户名"
 git config --global user.email "github邮箱"

  6. 在本机上生成密钥

     ssh-keygen -t rsa -c "github邮箱"

    公钥文件：~/.ssh/id_rsa.pub

    私钥文件：~/.ssh/id_rsa

    ~表示用户文件夹，Windows平台中指的是 C:/Users/用户名/

  7. 登录github，点击右上角头像下拉框-Settings-SSH and GPG keys-New SSH key,把刚刚生成密钥中的公钥内容复制进去，Title自起一个名字

  8. 检验密钥是否配置成功：右击空白处，打开Git Bash，输入下方代码第一行检验密钥，若全文显示如下则表示密钥配置成功

     $ ssh -T git@github.com
 Enter passphrase for key '/c/Users/王宇晨/.ssh/id_rsa':
 Hi Ice-Cocoa! You've successfully authenticated, but GitHub does not provide shell access.

  9. 使用git命令获取博客代码仓库

     $ git init  
 $ git remote add origin <代码仓库地址>
 $ git clone <代码仓库地址>

使用VScode代码编辑器编辑帖子内容，管理分支

  1. 安装VScode代码编辑器

  2. 使用VScode打开拉取的项目文件夹；或者使用VScode重新拉取一次代码仓库，打开这个项目文件夹

  3. 进行hexo主题配置，我的hexo-next主题个性化修改包括:

    1. 根目录下_config.yml配置：网页信息(title,author,copyright,language)，repo仓库地址配，本地搜索
    2. 主题目录下_config.yml配置：各类图标，显示方案和模式，侧边栏项目等
    3. 首页置顶配置，
    4. 首页只展示标签post
    5. 音乐外链配置
    6. 首页文章截取

  4. 打开文件夹下 /source/_posts/ ，在这里新建一个空的txt文件，重命名为 “帖子1.md” （名字随意，但后缀.md不能改变）

  5. 用VScode打开该文件，复制以下内容到.md文件内，并相应修改

     title: 帖子1
 categories:
     - 父类名称
     - 子类名称  #如不需要该子类可删除本行
     - 子类名称  #根据需要再新建子类
 tags: 展示    #设置任意值后，本贴会在首页展示，若不需要展示则在tags:后留空
 discription: #简短描述本帖意义或内容
 top: 0       #只有在首页展示top才有设置意义，取值0~100
                 #其中设置为1~100会置顶，同样的置顶文章，值越大越上面
 date: 2022-07-07 14:31:05  #帖子创建时间，不设定也不影响正常发帖
 ---
 前言部分，会在首页展示中预览
 <!-- more -->
 正文部分，需要使用Markdown语法编辑

  6. 进行本地部署、远程部署

  7. 右击项目文件夹打开git bash，运行以下代码，开启本地服务器

     hexo service # shell里可能需要开头加上npx，在shell开发人员设置里可设置

  8. 根据提示，在浏览器网址栏输入 http://localhost:4000/ ，完成了本地部署，现在在服务器运行期间在本地计算机查看网页

  9. 右击项目文件夹打开git bash，运行以下代码，部署到github页面仓库

     hexo clean
 hexo generate
 hexo deploy	#注意，这一步成功的前提是hexo已经配置完成
 #可能需要登录github
 #若配置密钥时设置密码，需要输入密码

  10. 在浏览器网址栏输入 github用户名.github.io , 若部署成功就可以查看博客页面

设置一旦触发推送博客就自动部署的功能

  1. 生成博客代码仓库和博客页面仓库的密钥

     ssh-keygen -f hexo-deploy-key -t rsa -C "github邮箱"

    以上代码分别在项目文件夹下生成 hexo-deploy-key私钥文件, hexo-deploy-key.pub公钥文件

  2. 在博客代码仓库 Setting-Secrets-Actions添加新密钥，使用hexo-deploy-key私钥文件内容配置

  3. 在博客页面仓库 Setting-Deploy Keys添加新密钥，使用hexo-deploy-key.pub公钥文件内容配置

  4. 在.github/workflows 目录下新建 deploy.yml 来编写GitHub Actions 工作流程，设置触发push就在docker里完成生成和部署的工作,注意文件内容在密钥名的适配
